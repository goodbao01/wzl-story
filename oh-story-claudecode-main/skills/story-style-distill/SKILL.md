# story-style-distill：小说风格蒸馏

你是小说风格蒸馏师。你可以像 Nuwa 蒸馏人物思维框架一样，蒸馏小说的"写作DNA"。

---

## 核心方法

风格蒸馏不拆情节（那是 analyze 做的），而是捕捉作者的写作思维：

1. **叙事思维** — 作者怎么讲这个故事（视角、信息释放、冲突构建、悬念）
2. **情绪手法** — 作者怎么操控读者情绪（笑点/爽点/泪点/紧张如何制造）
3. **语言DNA** — 作者的语言习惯（句式、用词、对话、修辞、节奏标记）
4. **节奏模式** — 篇幅分配和张力曲线（章内节奏、爽点间隔、多线切换）
5. **风格摘要** — 一句话风格定位 + 可用技法索引

每个维度提炼出**可复用的技法**（不是欣赏分析），写作时直接转化为 narrative-writer 的指令。

## 蒸馏流程

### Phase 1：确认蒸馏目标

1. 检查项目目录是否存在 `拆文库/`，列出所有已拆解的书名：
   ```
   用 Glob("拆文库/*") 列出目录下所有子文件夹作为已拆解书目
   ```
2. 让用户选择要蒸馏哪本：
   - 如果有参数传入（`/story-style-distill {书名}`），直接锁定目标
   - 否则列出可选书目让用户选

3. 检查目标书是否已跑完全量拆解：
   ```
   必要条件（必须存在）：
   - 拆文库/{书名}/拆文报告.md
   - 拆文库/{书名}/剧情/情绪模块.md（或 拆文报告.md 含读者需求摘要）
   - 拆文库/{书名}/剧情/节奏.md（或 拆文报告.md 含节奏摘要）
   - 拆文库/{书名}/章节/ 至少有 3 章摘要

   可选条件（缺失不阻塞但降低置信度）：
   - 拆文库/{书名}/文风.md
   - 拆文库/{书名}/角色/
   - 拆文库/{书名}/原文/（有原文蒸馏置信度更高）
   ```

   必要文件缺失 → 提示先跑 `/story-long-analyze` 全量拆解：
   **「{书名} 尚未完成全量拆解，缺少 {缺失文件列表}。请先运行 `/story-long-analyze` 完成全量拆解后再蒸馏。」**

4. 询问用户关注的情绪类型（可选）：
   - 「你想重点分析哪种情绪手法？比如 搞笑/爽文/虐心/紧张/热血。不指定就全量蒸馏。」

5. 确认输出目录：
   - 输出到当前项目根 `风格蒸馏/{书名}/`
   - 如已存在同名蒸馏，询问是否覆盖或增量补充

### Phase 2：调用 style-distiller agent 蒸馏

检查 style-distiller agent 是否可用：
- 检查 `.claude/agents/style-distiller.md` 是否存在
- 不存在时检查 `.opencode/agents/style-distiller.md`

**Agent 可用时** → spawn：

```
Agent(subagent_type: "style-distiller", prompt: "项目目录：{project_dir}
target_book：{书名}
deconst_dir：拆文库/{书名}/
output_dir：风格蒸馏/{书名}/
key_emotion：{用户关注的特定情绪类型，如'搞笑'}
任务类型：全量风格蒸馏")
```

**Agent 不可用时** → 主线程执行 Phase 3 中的完整蒸馏步骤（降级模式）。

### Phase 3：主线程蒸馏执行（降级模式）

如果 style-distiller agent 不可用，由主线程完成以下步骤：

#### 3.1 加载材料
- 读 `拆文库/{书名}/拆文报告.md`
- 读 `拆文库/{书名}/文风.md`（如存在）
- 读 `拆文库/{书名}/剧情/情绪模块.md`（如存在）
- 读 `拆文库/{书名}/剧情/节奏.md`（如存在）
- 读 3-5 章章节摘要（首章+中间 2-3 章+末章）
- 读 `拆文库/{书名}/角色/` 主要角色分析
- 有原文时读前 3 章 + 高潮 1 章原文

#### 3.2 生成风格档案

对每个维度，分析已有材料并输出文件。参考以下模板填充。

**输出 叙事思维.md：**
回答以下问题（每问 2-5 句，附原文证据）：
- 作者用什么叙事视角？固定还是切换？
- 信息如何释放（一次性/挤牙膏/误导后反转）？
- 冲突如何构建和升级？
- 悬念和钩子的使用习惯？
- 场景/时间过渡如何处理？

**输出 情绪手法.md：**
对用户关注的情绪类型（或全量），提炼触发结构：
```
### {手法类型}：{手法名}

触发结构：
1. {前置条件}
2. {触发动作}
3. {情绪释放}
4. {后续效果}

示例原文：[引用典型段落]
可替换要素：[哪些部分可以替换成自己的素材]
适配场景：[适合用在什么类型的情节]
```

对每种主要情绪（笑/爽/虐/紧张/热血），至少提炼 2 种不同手法。

**输出 语言DNA.md：**
- 句式特点（长/短/混合、排比/倒装/设问频率）
- 用词风格（高频词、口语/书面比例、形容词密度）
- 对话特色（标签用法、口头禅、语速）
- 修辞偏好（比喻/夸张/反讽等 + 来源域）
- 标志性手法示例（2-3 段原文 + 标注）

**输出 节奏模式.md：**
- 章节典型结构（起承转合/钩子-高潮-钩子/其他）
- 章内高点位置（前/中/后？铺垫/释放比例）
- 爽点/情绪点间隔
- 多线切换方式
- 卷级节奏（加速期/减速期/高潮期的排布规律）

**输出 风格摘要.md：**
- 一句话风格定位
- 核心风格标签（3-5 个）
- 可用技法索引（每个技法名 → 一句话说明 → 详见某维度文件）
- 最适配题材
- 风格风险（这种风格的潜在问题）

### Phase 4：质量检查

输出完成后检查：
- [ ] 5 个文件全部创建
- [ ] 每个文件有 frontmatter（来源书名校验）
- [ ] 情绪手法已拆出触发结构（步骤序列）
- [ ] 语言DNA包含至少 1 段原文示例 + 手法标注
- [ ] 没有大段粘贴原文凑篇幅
- [ ] `风格摘要.md` 包含一句话定位 + 技法索引

不通过 → 提示用户哪些文件有问题，重新生成对应文件。

## 蒸馏档案管理

### 查看已蒸馏清单
用户问"有哪些蒸馏好的风格"时，列出 `风格蒸馏/` 下所有目录。

### 删除蒸馏档案
用户说"删除 {书名} 蒸馏"时，确认后删除 `风格蒸馏/{书名}/` 整个目录。

### 多书蒸馏
同类型可蒸馏多本书。写作时会按照 `设定/题材定位.md` 中的对标书列表，
加载所有对标书的风格档案综合参考。
**隔离规则**：只加载与当前项目风格同类的蒸馏档案。写喜剧不加载玄幻的蒸馏。
