# 番茄小说平台适配指南

番茄小说（免费阅读/广告分成模式）的读者期望和写作禁忌。

## 平台特征

| 维度 | 特征 |
|------|------|
| 读者画像 | 下沉市场、碎片化阅读、手机端、年龄 16-30 |
| 付费模式 | 免费阅读（广告分成），依赖完读率和追读率 |
| 黄金章节 | 前 10 章决定留存，每章结尾决定是否继续 |
| 平台算法 | 完读率 > 追读率 > 评论互动 > 收藏 |

## 读者期望

| 期望 | 说明 |
|------|------|
| 开篇快 | 前 500 字必须出冲突或悬念，不能铺垫 |
| 每章有爽点 | 最少每章一个"爽"的情绪节点 |
| 章尾有钩子 | 每章结尾必须让读者想点下一章 |
| 节奏紧凑 | 少描写、少设定、多对话多冲突 |
| 更新稳定 | 日更 4000-6000 字是基础 |

## 写作禁忌

| 禁忌 | 说明 |
|------|------|
| 大段环境描写 | 超过 200 字的环境/外貌/背景说明会被跳过 |
| 慢热开头 | 前三章没进入正题 → 读者流失 |
| 复杂的设定说明 | 力量体系、世界观等通过对话和情节展示 |
| 文艺化表达 | 番茄读者不喜欢"散文感"，要直给 |
| 频繁切视角 | 手机阅读切视角容易晕 |
| 无爽点章节 | 没有爽点的章节完读率极低 |

## 字数建议

| 节奏 | 建议字数 | 说明 |
|------|----------|------|
| 正常章 | 2000-3000 字 | 手机阅读舒适区间 |
| 高潮章 | 2500-3500 字 | 可略长，但不要超过 4000 |
| 过渡章 | 1500-2000 字 | 保证有钩子即可 |

## 写入 prompt 的平台指令

```markdown
### 番茄平台适配
- 开篇 500 字内必须有冲突/悬念
- 每章至少一个显性爽点
- 章尾必须有强钩子（悬念/问题/反转预告）
- 减少环境描写和背景说明，用对话和行动展示
- 全景描述不超过 200 字
- 控制章节长度在 2000-3000 字
- 少用文艺修饰词，叙事直给
```
